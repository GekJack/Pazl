# Pazl
## Опис завдання
### Програма була розроблена для розв'язання задачі складання "пазлів". Умови:

Кожна частина пазлу має "початок" і "кінець" (2 символи).\
Частини потрібно з'єднати так, щоб кінець однієї збігався з початком іншої.\
Необхідно знайти найдовше можливе з'єднання всіх частин пазлу.

## Використаний підхід
### Постановка задачі
Завдання було розбито на етапи: 

- Аналіз вхідних даних: файл із числами довжиною 6 символів. 
- Перевірка коректності даних.
- Алгоритм з'єднання частин пазлів, використовуючи збіги "кінців".
- Пошук найдовшого результату.

#### Алгоритм рішення
 - Перевірка існування файла за вказаним шляхом (*IsFileAccessible*)
 - Перевірка коректності даних у вказаному файлі та отримання значень з файлу.(За довжиною, яка дорівнює 6, та на відповідність рядків числам. Метод - CheckAllPiecesIsNumAndGetValues)
 - Рекурсивний пошук шляхів: Алгоритм реалізований у методі AlgorithmFindPuzzle. Він рекурсивно з'єднує частини пазлів у двох напрямках (зліва і справа), уникаючи повторного використання тієї ж частини.
 - Визначення найдовшого рішення(FindTheLongest).

#### Основні принципи
- Оптимізація роботи з пам'яттю: Для зберігання даних використовуються словники (Dictionary) і списки (List), що дозволяє швидко знаходити й видаляти елементи.
- Рекурсія: Рекурсивний підхід використовується для побудови всіх можливих комбінацій з'єднань.
## Опис методів програми
### Main
Точка входу в програму, яка отримує дані від користувача, перевіряє коректність введених даних, організовує обробку файлу з частинами пазлу, запускає алгоритм пошуку можливих рішень і виводить фінальний результат у вигляді найдовшого зібраного пазлу.
### AlgorithmFindPuzzle
Метод AlgorithmFindPuzzle реалізує рекурсивний пошук можливих комбінацій для складання пазлу. Він обирає частину пазлу для з'єднання, визначає напрямок з'єднання (праворуч або ліворуч) і перевіряє можливість використання кожного елемента, уникаючи повторного використання. У процесі пошуку метод формує нові комбінації та зберігає їх у тимчасовому списку. Завдяки рекурсії метод знаходить всі можливі варіанти складання пазлу.
### ChangeKey
Зміна ключа словника для даних, які були вже використані на цьому рівні рекурсії.
### DictionaryGeneration
Генерує словник з фільтрацією початкових значень за початком та кінцем числа.
### CheckAllPiecesIsNumAndGetValues
Перевіряє всі фрагменти пазла на відповідність числу та довжину, записує їх у відповідний список для подальшої обробки
### IsFileAccessible
Перевірка файла на доступність та існування за вказаним шляхом.
### ConnectingPuzzlePartLeft
З'єднання фрагментів пазла ліворуч до основної частини.
### ConnectingPuzzlePartRight
З'єднання фрагментів пазла праворуч до основної частини.
### FindTheLongest
Знаходження в списку готових варіацій пазла найдовшого рішення.
## Опис основних структур даних
### sourceFilePath
Шлях до файлу з вхідними даними.
### source_puzzle_pieces
Список всіх початкових фрагментів пазла.
### puzzle
Список для зберігання отриманих найдовших варіацій пазла з кожной гілки рекурсії.
### puzzle_temp
Список для зберігання отриманих найдовших варіацій пазла з однієї гілки рекурсії.
### source_puzzle_filtered_in_parts_to_connect
Словник для зберігання відфільтрованих за початком та кінцем вхідних даних.
### used_puzzle_parts
Словник для зберігання використаних фрагментів, задля уникання повторів.
## Використані технології
- Мова програмування: C#.
- Платформа: .NET 8
- Середовище розробки: Visual Studio 2022
- Інструменти роботи з файлами: System.IO для читання та перевірки доступу до файлів.
